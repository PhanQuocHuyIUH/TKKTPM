# Bài 7: Biến Môi Trường trong Dockerfile

## Mô tả
Ứng dụng Python đọc biến môi trường `APP_ENV` và in ra màn hình. Dockerfile định nghĩa giá trị mặc định bằng `ENV`, nhưng giá trị này có thể bị ghi đè khi chạy container.

## Cấu trúc file
```
bai7-envvar/
├── Dockerfile
├── app.py
└── README.md
```

## Giải thích Dockerfile
| Lệnh | Ý nghĩa |
|------|---------|
| `FROM python:3.9` | Base image Python 3.9 |
| `WORKDIR /app` | Thư mục làm việc |
| `ENV APP_ENV=development` | Khai báo biến môi trường với giá trị mặc định |
| `COPY app.py .` | Copy script Python |
| `CMD ["python", "app.py"]` | Chạy script |

## Hướng dẫn chạy

### Bước 1: Build Docker Image
```bash
docker build -t hello-envvar .
```

### Bước 2a: Chạy với giá trị ENV mặc định (development)
```bash
docker run --rm hello-envvar
```
**Kết quả:** `Application is running in: development mode`

### Bước 2b: Ghi đè biến môi trường khi chạy
```bash
docker run --rm -e APP_ENV=production hello-envvar
```
**Kết quả:** `Application is running in: production mode`

```bash
docker run --rm -e APP_ENV=staging hello-envvar
```
**Kết quả:** `Application is running in: staging mode`

### Bước 2c: Truyền biến từ file .env
Tạo file `.env`:
```
APP_ENV=testing
```
Chạy:
```bash
docker run --rm --env-file .env hello-envvar
```
**Kết quả:** `Application is running in: testing mode`

## Các cách khai báo biến môi trường

| Phương thức | Ví dụ | Ưu tiên |
|-------------|-------|---------|
| `ENV` trong Dockerfile | `ENV APP_ENV=development` | Thấp nhất |
| `-e` khi `docker run` | `-e APP_ENV=production` | Cao hơn |
| `--env-file` | `--env-file .env` | Cao hơn |

> Biến truyền vào lúc `docker run` luôn ghi đè giá trị `ENV` trong Dockerfile.
