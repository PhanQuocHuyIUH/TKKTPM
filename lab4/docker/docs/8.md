# Bài 8: PostgreSQL Tùy Chỉnh với Docker

## Mô tả
Dockerfile chạy PostgreSQL 15 với database, user, và bảng được tạo tự động từ file `init.sql` khi container khởi động lần đầu.

## Cấu trúc file
```
bai8-postgres/
├── Dockerfile
├── init.sql
└── README.md
```

## Giải thích Dockerfile
| Lệnh | Ý nghĩa |
|------|---------|
| `FROM postgres:15` | Base image PostgreSQL phiên bản 15 |
| `ENV POSTGRES_DB=labdb` | Tên database sẽ được tạo tự động |
| `ENV POSTGRES_USER=labuser` | Tên superuser của database |
| `ENV POSTGRES_PASSWORD=labpassword` | Mật khẩu cho user |
| `COPY init.sql /docker-entrypoint-initdb.d/` | Script SQL tự động chạy lần đầu khởi động |

> **Cơ chế hoạt động:** Postgres image tự động chạy tất cả file `.sql` và `.sh` trong thư mục `/docker-entrypoint-initdb.d/` khi data directory trống (lần đầu tiên).

## Hướng dẫn chạy

### Bước 1: Build Docker Image
```bash
docker build -t custom-postgres .
```

### Bước 2: Chạy Container
```bash
docker run -d -p 5432:5432 --name postgres-container custom-postgres
```

### Bước 3: Kết nối vào PostgreSQL
**Dùng psql bên trong container:**
```bash
docker exec -it postgres-container psql -U labuser -d labdb
```

**Kiểm tra bảng và dữ liệu:**
```sql
\dt
SELECT * FROM users;
SELECT * FROM products;
```

### Bước 4: Kết nối từ máy host (nếu có psql cài sẵn)
```bash
psql -h localhost -p 5432 -U labuser -d labdb
```

### Dừng và xóa container
```bash
docker stop postgres-container
docker rm postgres-container
```

## Lưu ý quan trọng
- Script `init.sql` **chỉ chạy một lần** khi data directory còn trống.
- Nếu muốn chạy lại init.sql, phải xóa volume dữ liệu.
- Trong production, nên dùng **Docker Secrets** thay vì `ENV` để bảo mật mật khẩu.

## Dùng Volume để giữ dữ liệu
```bash
docker run -d \
  -p 5432:5432 \
  -v postgres_data:/var/lib/postgresql/data \
  --name postgres-container \
  custom-postgres
```
